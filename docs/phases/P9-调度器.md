## 详细任务

### T1. `services/scheduler.py` — SchedulerService

```python
class SchedulerService:
    def __init__(self, task_log_repo, schedule_store, timezone="Asia/Shanghai"): ...
    async def start(self):
        """启动 AsyncIOScheduler，从 schedule_store 加载所有 enabled 任务并注册。"""
    async def stop(self):
        """停止并清理所有任务。"""
    def register(self, task: ScheduleTask, callback: Callable):
        """注册一个调度任务，cron 触发时调 callback(task)。"""
    def unregister(self, task_id: str): ...
    async def reload(self):
        """热重载：停全部 → 重新从 store 加载 → 重启。配置变更时调用。"""
    async def run_now(self, task_id: str):
        """立即触发一次（跳过 cron 等待）。"""
    async def execute_task(self, task: ScheduleTask, callback):
        """执行单个任务：create task_log(running) → callback → update(success/error)。
        统一记录 start/end/duration/status/result_count/message。"""
```

核心：
- 用 `apscheduler.schedulers.asyncio.AsyncIOScheduler`。
- 触发器：`CronTrigger.from_crontab(task.cron, timezone=...)`。
- 任务存储：内存（`MemoryJobStore`），状态持久化在 schedule_store（EntityJsonRepository 的 schedules 表）。
- task_log：每次执行 create 一条；callback 可返回 `{"result_count": int, "message": str}`，记入 log。
- 异常容错：callback 抛异常 → task_log status=error + message=str(e)，不影响调度器本身。
- 启动修复在 P2 已做（running→interrupted）。

**回调注入设计**（关键）：
- 调度器不知道 task_type 怎么执行。提供 `TaskExecutorRegistry`：`task_type → async callback(task) -> dict` 的映射。
- P11 注册 `daily_digest` 的 executor；P14 注册 `full_ingestion`/`adapter` 的 executor。
- `execute_task` 从 registry 取对应 callback；无映射则记 error。

### T2. 测试

- `tests/services/test_scheduler.py`：
  - register + run_now：mock callback 被调用，task_log 记录正确（running→success，含 duration/result_count）。
  - callback 抛异常：task_log status=error，调度器不崩。
  - unregister：任务不再触发。
  - reload：重新加载后任务列表正确。
  - schedule CRUD（save/get/delete 经 store）。
  - task_type 无 executor：run_now 记 error。
  - cron 解析：合法 cron 注册成功；非法 cron 报错（不注册）。
