## 详细任务

### T1. `renderers/models.py` — 共享数据模型

```python
@dataclass(frozen=True)
class DigestItem:  # 若 P7 已定义，import 即可
    title: str; summary: str; url: str; source: str; score: float | None = None
    original_id: str | None = None

@dataclass(frozen=True)
class CuratedDigest:
    date: str                     # ISO 日期
    title: str                    # 如 "每日精选 · 2026-07-15"
    items: list[DigestItem]
    summary: str = ""             # 整体概述
    total_scanned: int = 0        # 抓取总数（精选前的量）
```

> **核查**：若 P7 的 `DigestItem` 字段不同，停下问或在本 models.py 统一定义并让 P7/P8 import（在 review 说明）。

### T2. `agents/pipelines/prompts.py` — 精选 prompt

- `CURATE_PROMPT`：让 LLM 对一批资讯打分（重要性 1-10）+ 相关性 + 生成中文摘要（50-100字），返回严格 JSON。
  - 输入：今日 UnifiedData 列表（title+description+source）。
  - 输出 JSON：`[{"id","title","summary","score","score_reason"}, ...]`。
  - 用 `digest.md` 模板的 `## [curate]` section（P4 已建文件）。
- `DIGEST_OVERVIEW_PROMPT`：对 top-N 生成一段整体概述（可选）。

### T3. `services/publishing.py` — PublishingService

```python
class PublishingService:
    def __init__(self, publisher_registry, renderers: dict[str, Callable]): ...
    async def fanout(self, digest: CuratedDigest, targets: list[str]) -> dict[str, dict]:
        """并行推送到多个目标端。
        targets: ["feishu_bot","wecom_bot"]（publisher id 列表）。
        每端：renderer 生成 content → publisher.publish。
        并行 asyncio.gather(return_exceptions=True)，单端失败不影响其他。
        返回 {target_id: {status, error?}}。
        """
```

- renderer 映射：`feishu_bot` → `render_digest_card`；`wecom_bot` → `render_digest_markdown`。
- 限流：端间并行，但同端串行（防止企微 20/min 限流，本 MVP 单次推送一条无虞，注释说明）。

### T4. `agents/pipelines/daily_digest.py` — 流水线（DAG workflow + executor）

这是核心。**定义为 DAG 工作流**（复用 P10 引擎，体现价值）：

```python
def build_daily_digest_workflow(config: DailyDigestConfig) -> WorkflowDefinition:
    """构造每日推送工作流：
    step1 ingest     → adapter 抓取
    step2 dedupe     → 去重（url/sha256 跨日）
    step3 curate     → AI 打分精选 [LOOP 节点: max_iterations=3, exit_condition=llm]
    step4 overview   → 整体概述（可选）
    step5 render+fanout → 渲染 + 推送
    各步通过 input_map 声明数据依赖。
    """
```

各步骤实现（作为 inline 函数或注册成可被引擎调用的 step executor）：

1. **ingest**：调 `IngestionService.run_all`（或指定 adapter），返回 `list[UnifiedData]`。
2. **dedupe**：按 url 归一化 + content sha256 去重；按 published_date 过滤（只取最近 N 天，config.fetch_days）。
3. **curate（Loop 节点）**：
   - 输入：去重后的 UnifiedData 列表。
   - agent 执行 `CURATE_PROMPT`，解析 JSON → 每条带 score + summary。
   - 排序取 top-N（config.top_n，默认 5）。
   - **Loop**：`Reflector.assess("精选质量", curate_output)`；不达标（如评分含低质量摘要）带反馈重生成。max_iterations=3。
   - 输出：`list[DigestItem]`。
4. **overview（可选）**：对 top-N 生成一句话概述。
5. **render+fanout**：组装 `CuratedDigest` → `PublishingService.fanout` → 多端。

**DailyDigestConfig**：`adapter_ids: list[str]`, `fetch_days: int=2`, `top_n: int=5`, `targets: list[str]=["feishu_bot","wecom_bot"]`, `enable_overview: bool=True`, `curate_agent_id: str`, `loop_max_iterations: int=3`。

**executor（注册到 P9 TaskExecutorRegistry）**：
```python
async def daily_digest_executor(task: ScheduleTask) -> dict:
    """从 task.config 取 DailyDigestConfig → 调 WorkflowEngine.run → 返回 {result_count, message}。"""
```

## 验收条件

1. 流水线作为 DAG 工作流正确编排（5 节点，input_map 数据依赖，引擎能跑）。
2. AI 精选产出正确 JSON，top_n 截断，评分排序。
3. **Loop 自评节点工作**：差质量时带反馈重生成，达 max_iterations 退出（测试可见 loop_iteration）。
4. 去重生效（url/sha256）。
5. fanout 并行多端，单端失败容错。
6. JSON 解析有兜底（非法 JSON 不崩）。
7. daily_digest_executor 注册到 TaskExecutorRegistry，可被调度器触发。
8. 全测试绿，零真实网络。
