## 详细任务

### T1. `renderers/wecom_markdown.py`

企业微信群机器人支持的 markdown 语法**受限**（不支持图片 base64，链接语法特定）。按企微规范渲染：

```python
def render_digest_markdown(title: str, items: list[DigestItem], *, footer: str | None = None) -> str:
    """返回企微兼容的 Markdown 字符串。

    结构示例：
    ## 📰 每日精选 · 2026-07-15

    **1. <标题>**
    > <摘要>
    > [来源](<url>) · 评分 8.5

    **2. ...**
    ...
    """
```

- DigestItem 复用 P7 定义（从 `renderers/feishu_card` import DigestItem，或抽到 `renderers/models.py`——**P7 若已定义在 feishu_card，本包 import 即可，避免重复**）。
- 企微 markdown 注意：链接用 `[文本](url)`；标题用 `#`/`##`；引用 `>`；**不支持** 表格/图片 base64。
- 提供文本降级 `render_digest_plain(title, items) -> str`。

### T2. `plugins/builtin/publishers/wecom.py` — WeComPublisher

```python
class WeComPublisher(BasePublisher):
    metadata = PluginMetadata(
        id="wecom_bot", type=PluginType.PUBLISHER, name="企业微信机器人",
        description="通过群机器人 Webhook 推送到企业微信群",
        icon="wecom",
        config_fields=[
            ConfigField(key="webhook", label="Webhook Key", type="text", required=True, scope="adapter",
                        help_text="企微 webhook URL 的 key 部分，或完整 URL"),
        ],
    )
    async def publish(self, content, options=None) -> dict:
        """content 为 markdown str 或 plain str；options 含 webhook。"""
```

核心逻辑：
1. webhook：options.webhook 可以是完整 URL 或 key（key 时拼 `https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=<key>`）。
2. payload：
   - markdown：`{"msgtype":"markdown","markdown":{"content": <md>}}`。
   - text：`{"msgtype":"text","text":{"content": <txt>}}`。
3. POST（httpx，超时 10s）。
4. 企微返回 `{"errcode":0,"errmsg":"ok"}` → 成功；errcode != 0 抛 `PublisherError`（带 errmsg）。
5. 重试：指数退避（最多 3 次）。
6. 限流：企微单机器人 20 条/分钟，publisher 内部不做限流（由调用方 P11 流水线控制频率），但注释说明。

**可选 msgtype 扩展**（MVP 实现 markdown + text 即可，news 图文后置）。
