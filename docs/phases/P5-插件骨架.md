## 详细任务

### T1. `plugins/base.py` — 四类抽象基类

严格按 `docs/conventions/plugin-contract.md` 实现：
- `PluginMetadata`/`ConfigField`（如 P1 已定义则 import，不重复定义——**核对后若 P1 未含则在 domain/models.py 补，但本包优先 import**）。
- `class BaseAdapter(ABC)`：`metadata: ClassVar[PluginMetadata]`；abstract `async fetch(config)`、`transform(raw, config)`；实现的 `async fetch_and_transform(config)` 模板方法（fetch→transform，错误返回 `[]`）。
- `class BasePublisher(ABC)`：abstract `async publish(content, options)`；可选 `async get_item_url(item)`。
- `class BaseStorageProvider(ABC)`：abstract `async upload(local_path, target_path) -> str | None`。
- `class BaseTool(ABC)`：`id/name/description/parameters: ClassVar`；abstract `async handler(args)`。

> **核查项**：开工前确认 `PluginMetadata`/`ConfigField`/`PluginType` 是否已在 `domain/models.py`。若无，本包在 `plugins/base.py` 定义并让 domain import 反向（但更优是 P1 已有——若没有，在 review 里标注并按 P1 契约补到 domain）。

### T2. `plugins/registry.py`

- `class AdapterRegistry`、`PublisherRegistry`、`StorageRegistry`：各为单例（模块级 `_instance` + `get_instance()`），内部 `dict[str, tuple[type, PluginMetadata]]`。
  - `register(key, cls, metadata)`、`get(key) -> type`、`get_metadata(key)`、`list() -> list[type]`、`list_metadata() -> list[PluginMetadata]`。
- `class ToolRegistry`：双注册——`register(cls, metadata=None)`（类）+ `register_tool(tool: BaseTool)`（实例）；`get_tool(id) -> BaseTool`、`async call_tool(id, args)`、`get_all_tools() -> list[ToolDefinition]`、`list_metadata()`。

### T3. `plugins/discovery.py` — 自动发现

- `def scan_and_register(package_root: str | None = None)`：
  - 用 `importlib` + `pkgutil.walk_packages` 扫描 `plugins/builtin/**` 与 `plugins/custom/**`（custom 目录不存在则跳过）。
  - 跳过文件名含 `base`/`_base` 的模块、`__init__`。
  - 对每个模块遍历 `dir(module)`，取 class，若 `hasattr(cls, 'metadata')` 且 `isinstance(metadata, PluginMetadata)` 则按 `metadata.type` 注册到对应 Registry。
  - Tool 额外：注册类后**不立即实例化**（实例化留给 bootstrap，因为要注入依赖）。
  - 返回 `DiscoveryResult`（注册了哪些，跳过了哪些）。
- 幂等：重复 scan 不重复注册（按 key 覆盖）。

### T4. `plugins/builtin/tools/execute_command.py` — 示例 Tool

- `class ExecuteCommandTool(BaseTool)`：`id="execute_command"`，`name="execute_command"`，parameters JSON Schema（command: str required, cwd: str optional, timeout: int optional）。
- 安全：`ALLOWED = {"node","npm","npx","git","ls","echo","python","pip","uv"}` 自动批准；`BLOCKED = {"rm","format","mkfs","dd","shutdown"}` 拒绝。
- `async handler(args)`：解析命令首词，黑名单拒绝抛 `ToolExecutionError`；白名单直接 `asyncio.create_subprocess_shell` + 超时；非白名单可返回需审批提示（MVP 直接拒绝或返回 pending，注释后续接审批）。
- 截断 stdout/stderr（防止超长）。

### T5. 测试

- `tests/plugins/test_base.py`：基类不能直接实例化（ABC）；子类化后必须实现 abstract 方法；fetch_and_transform 错误返回 `[]`。
- `tests/plugins/test_registry.py`：register/get/list_metadata；单例；ToolRegistry 双注册 + call_tool；重复 register 覆盖。
- `tests/plugins/test_discovery.py`：用临时包目录造几个带/不带 metadata 的假类，scan 后正确注册带 metadata 的、跳过不带的、跳过 base 文件。
- `tests/plugins/test_execute_command.py`：白名单命令执行成功；黑名单拒绝；超时处理；输出截断。

## 验收条件

1. 四基类 + 四 Registry + 自动发现实现完整，符合 plugin-contract.md。
2. 自动发现能正确识别带 metadata 的类、跳过基类与非插件。
3. ExecuteCommandTool 白黑名单生效（黑名单拒绝，白名单执行）。
4. ToolRegistry 双注册工作（register 类 + register_tool 实例 + call_tool）。
5. mypy strict + ruff + pytest 全绿。
