# P0.7 — API 中转端点支持（Custom Base URL）

> **状态**：未开始 · **前置依赖**：P0.6 已通过

## 目标

将 `.env` 中的自定义 API 端点（`OPENAI_API_BASE_URL` / `ANTHROPIC_API_BASE_URL`）映射到对应的 `ProviderConfig.base_url`，使第三方中转（如 `https://api.ej2075.com/v1`）可直接配置。

## 前置依赖

P0.6 已通过。

## 根因分析

```
.env: OPENAI_API_BASE_URL=https://api.ej2075.com/v1
     ↘
ProviderConfig[id=default-openai].base_url  ← 从未映射，始终为空
     ↘
ChatOpenAI(base_url=None) → 使用官方 https://api.openai.com
     ↘
国内直连 → 100% 失败
```

`ProviderConfig.base_url` 字段已存在（docstring 明确写了 "may serve multiple models; one api_key + base_url"），且 OpenAIProvider / AnthropicProvider 已正确传递 `base_url=config.base_url or None`，只是 `SystemSettings` 没有把它从 `.env` 读出来。

## 可改范围（白名单）

- `src/multiscribe_agent/config.py`（仅限以下字段：在 `SystemSettings` 中追加 `openai_api_base_url` 和 `anthropic_api_base_url` 字段 + 对应 `model_validator` 绑定逻辑）
- `.env.example`（追加 `OPENAI_API_BASE_URL=` / `ANTHROPIC_API_BASE_URL=` 注释示例）
- `tests/test_base_url_binding.py`（新增：验证 base_url 别名绑定，不修改现有测试）

## 禁止改动（黑名单）

- 不修改 `llm/providers/`（base_url 透传已正确实现，本次无需改动）。
- 不修改 `bootstrap.py`（P0.6 已完成 proxy 透传，本次无需改动）。

## 详细任务

### T0. `config.py` — base_url 字段与绑定

#### 0a. 新增字段声明

在 `SystemSettings` 中追加两个 `AliasChoices` 字段（放在 `anthropic_api_key` 附近）：

```python
openai_api_base_url: str = Field(
    default="",
    validation_alias=AliasChoices(
        "OPENAI_API_BASE_URL",
        "MULTISCRIBE_OPENAI_API_BASE_URL",
    ),
)
anthropic_api_base_url: str = Field(
    default="",
    validation_alias=AliasChoices(
        "ANTHROPIC_API_BASE_URL",
        "MULTISCRIBE_ANTHROPIC_API_BASE_URL",
    ),
)
```

#### 0b. 扩展 model_validator 绑定逻辑

在已有的 `_bind_mvp_environment_values` 中追加：

```python
# 在 api_keys 循环之后、publisher 循环之前追加：
for provider in self.ai_providers:
    if provider.type == "openai" and self.openai_api_base_url:
        provider.base_url = self.openai_api_base_url
    if provider.type == "anthropic" and self.anthropic_api_base_url:
        provider.base_url = self.anthropic_api_base_url
```

> **注意**：`openai_api_base_url` 和 `openai_api_key` 是独立的两个字段——前者填中转地址，后者填 API key。两者可以同时填写，也可以只填其一。

### T1. `.env.example` — 追加端点示例

在 `OPENAI_API_KEY=` 后追加：

```bash
# 中转端点（选填。与 API key 配套使用）
# 设置后 OpenAI provider 将请求 https://api.ej2075.com/v1 而非官方地址
OPENAI_API_BASE_URL=
# 同上，Anthropic 中转地址
ANTHROPIC_API_BASE_URL=
```

### T2. 测试 `tests/test_base_url_binding.py`

- 测试 `OPENAI_API_BASE_URL=https://api.ej2075.com/v1` → `providers[id=default-openai].base_url`

## 使用示例

你的 `.env` 可以这样配置中转：

```bash
# 使用第三方中转（ej2075.com）
OPENAI_API_KEY=sk-你的key
OPENAI_API_BASE_URL=https://api.ej2075.com/v1

# 或者使用 Anthropic 中转
ANTHROPIC_API_KEY=sk-ant-你的key
ANTHROPIC_API_BASE_URL=https://api.ej2075.com/anthropic/v1

# 代理（可选，取决于中转是否需要）
HTTP_PROXY=

# 推送
FEISHU_WEBHOOK=https://open.feishu.cn/open-apis/bot/v2/hook/...
```

> **中转 vs 代理的区别**：HTTP_PROXY 转发所有 TCP 流量；`OPENAI_API_BASE_URL` 只改变 OpenAI/Anthropic 的目标地址，两者可以叠加使用。
