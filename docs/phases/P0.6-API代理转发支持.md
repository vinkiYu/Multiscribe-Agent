# P0.6 — API 代理转发支持（Proxy Forwarding）

> **状态**：未开始 · **前置依赖**：P12 已通过 · **阻塞**：无

## 目标

将 `SystemSettings.http_proxy` 穿透到 LLM provider 调用，使国内服务器可通过代理（clash、v2ray 等）访问 OpenAI / Anthropic API。

## 前置依赖

P12 已通过。

## 根因分析

```
SystemSettings.http_proxy = "http://127.0.0.1:7890"  ← 已存在
    ↘
bootstrap._provider_for_agent()    ← 从未读取 http_proxy
    ↘
create_provider(provider, model, temperature)   ← proxy 未传递
    ↘
OpenAIProvider.__init__(proxy=None)  ← httpx AsyncClient 无 proxy
AnthropicProvider.__init__(proxy=None)  ← ChatAnthropic 无 anthropic_proxy
    ↘
真实请求直连境外 API → 国内环境 100% 失败
```

`ProviderConfig.use_proxy` 是另一个可选开关（P1 预留），但本次修复优先打通最简路径：全局 `http_proxy` → 所有 provider。

## 可改范围（白名单）

- `src/multiscribe_agent/bootstrap.py`（在 `_provider_for_agent()` 中读取 `settings.http_proxy` 并传入 `create_provider()`）
- `tests/test_proxy_routing.py`（新增：验证 proxy 参数透传，不修改现有测试）

## 禁止改动（黑名单）

- 不修改 `ProviderConfig` / `ProviderConfig.use_proxy` 字段（现有设计，P1 预留字段，本次不动）。
- 不修改 `llm/provider.py` 和 `llm/providers/`（已正确支持 proxy 参数，本次无需改动）。
- 不修改 `config.py`（`http_proxy` 字段已存在，无需改动）。

## 详细任务

### T1. `bootstrap.py` — proxy 穿透

在 `_provider_for_agent()` 中，将 `self.settings.http_proxy` 传入 `create_provider()`：

```python
# 现有（不变）：
return create_provider(provider, model=definition.model, temperature=definition.temperature)

# 修改为（添加 proxy）：
return create_provider(
    provider,
    model=definition.model,
    temperature=definition.temperature,
    proxy=self.settings.http_proxy or None,
)
```

`or None` 保证空字符串不传入（httpx 和 ChatAnthropic 对 `None` proxy 有明确处理）。

### T2. 测试 `tests/test_proxy_routing.py`

- mock `create_provider` 调用，验证 `proxy` 参数等于 `settings.http_proxy`
- 验证 `http_proxy=""` 时 `proxy=None` 传入（不传入空字符串）
- 验证多个 agent 定义共享同一 provider 时 proxy 一致透传

---

## 验收条件

1. `bootstrap._provider_for_agent()` 调用 `create_provider()` 时传入 `proxy=settings.http_proxy or None`。
2. 空 `http_proxy` 不传入空字符串（`None` 而非 `""`）。
3. 全测试绿（不含真实网络）。

## 与 P13 的衔接

P0.6 通过后，你的 `.env` 可以这样配置代理：

```bash
# 代理地址（clash 默认端口 7890，v2ray 默认 10808）
HTTP_PROXY=http://127.0.0.1:7890

# LLM API Key
OPENAI_API_KEY=sk-...
```

国内服务器只需填 `HTTP_PROXY`，API key 直通 OpenAI/Anthropic 官方地址，无需改任何其他配置。
