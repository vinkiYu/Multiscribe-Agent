## 一、问题定位

### P30 验收 7 深度测试结果

P30 的 `_curate_item_dict` 把 `description` 截断到 500 字符作为 summary，保留 6 个字段。ZCode 用 8 种真实业务场景独立测试，结果：

| 场景 | description 特征 | 精简比例 | < 30%？ |
|---|---|---:|:---:|
| RSS 短摘要（35 字） | 最常见 | **49.5%** | ❌ |
| RSS 中等+HTML（70 字） | 常见 | **53.1%** | ❌ |
| GitHub README（150 字） | 常见 | **62.0%** | ❌ |
| AI 搜索超长正文（1500 字） | 少见 | **70.0%** | ❌ |
| 极短 ASCII（35 字） | 少见 | **50.4%** | ❌ |
| **混合分布（最接近真实）** | 真实业务 | **58.7%** | ❌ |
| 30 条少量候选 | — | **49.5%** | ❌ |
| metadata 巨大（2000 字） | 极端 | 4.9% | ✅ |

**通过率：1/8**。只有 metadata 极端膨胀的场景才达标。

### 根因

```
每条旧体积(model_dump) ≈ description + metadata(~80) + author(~30) + status(~15) + 其他(~50) ≈ description + 175
每条新体积(当前投影)  ≈ min(description,500) + id + title + url + source + category + JSON结构开销 ≈ description_capped + 135
```

当 description < 500 时（即绝大多数 RSS 场景），截断不生效，新旧体积差只有 ~40 字符/条，**精简比例 50-70%**。

### 方案对比（ZCode 实测）

| 方案 | summary 上限 | 保留字段 | 混合场景比例 | 6 场景通过数 |
|---|---:|---|---:|:---:|
| 当前实现 | 500 | id/title/summary/url/source/category | 58.7% | 0/6 |
| v1 | 200 | id/title/summary/category | 34.0% | 0/6 |
| **v2（推荐）** | **150** | **id/title/summary** | **27.8%** | **3/6** |
| v3 | 150 | 管道分隔(id\|title\|summary) | 20.1% | 6/6 |
| v4 | 100 | id/title/summary | 25.9% | 3/6 |

**v3 全部通过但牺牲了 JSON 结构化**，LLM 解析管道分隔文本有风险。

**v2 是最优解**：保留 JSON 结构化，summary 截断到 150 字符（LLM 打分排序足够），去掉 url/source/category（curate 阶段不需要）。v2 在短内容场景（35 字 RSS）能达 27.0%，混合场景 27.8%，均 < 30%。

### 为什么 LLM 不需要 url/source/category？

代码确认（`daily_digest.py:393-410`）：
- LLM 返回 `id/title/summary/score/score_reason`。
- 代码通过 `by_id = {item.id: item for item in items}` 用 `id` 回映到原始 `UnifiedData`。
- `url` 和 `source` 是回映后才从原始对象取的，**LLM 在 curate 阶段不需要看到它们**。
- `category` 对打分排序没有帮助。

```python
# src/multiscribe_agent/agents/pipelines/daily_digest.py

# 旧：
_CURATE_SUMMARY_CHAR_LIMIT = 500

def _curate_item_dict(item: UnifiedData) -> dict[str, object]:
    """Project normalized content to the bounded fields needed for curation."""
    return {
        "id": item.id,
        "title": item.title,
        "summary": item.description[:_CURATE_SUMMARY_CHAR_LIMIT],
        "url": item.url,
        "source": item.source,
        "category": item.category,
    }

# 新：
_CURATE_SUMMARY_CHAR_LIMIT = 150


def _curate_item_dict(item: UnifiedData) -> dict[str, object]:
    """Project normalized content to the minimal fields needed for curation scoring.

    Only id (for result mapping), title (topic), and summary (content excerpt)
    are needed by the LLM to rank and summarize candidates. url, source, and
    category are recovered from the original UnifiedData via by_id lookup
    after curation, so they are not included here to minimize prompt size.
    """
    return {
        "id": item.id,
        "title": item.title,
        "summary": item.description[:_CURATE_SUMMARY_CHAR_LIMIT],
    }
```

**改动要点**：
1. `_CURATE_SUMMARY_CHAR_LIMIT`：500 → 150。
2. 去掉 `url`/`source`/`category` 三个字段。
3. docstring 说明为什么只需要 3 个字段（回映机制）。

**新增** `tests/agents/pipelines/test_curate_projection_fields.py`：

```python
"""Verify _curate_item_dict projection fields, truncation, and ratio across scenarios."""

import json
import pytest
from multiscribe_agent.domain.models import UnifiedData
from multiscribe_agent.agents.pipelines.daily_digest import (
    _curate_item_dict,
    _CURATE_SUMMARY_CHAR_LIMIT,
)


def _make_item(description: str, idx: int = 0) -> UnifiedData:
    return UnifiedData(
        id=f"item-{idx}",
        title=f"标题{idx}",
        url=f"https://example.com/{idx}",
        description=description,
        published_date="2026-07-24",
        source="rss",
        category="tech",
        metadata={"tags": ["AI"], "score": idx},
        author=f"author-{idx}",
        status="active",
    )


# 验收 1: 只返回 3 个字段
def test_projection_returns_only_id_title_summary():
    item = _make_item("一段描述文字")
    projected = _curate_item_dict(item)
    assert set(projected.keys()) == {"id", "title", "summary"}


# 验收 2: summary 截断到 150
def test_summary_is_truncated_to_limit():
    assert _CURATE_SUMMARY_CHAR_LIMIT == 150
    long_desc = "x" * 500
    item = _make_item(long_desc)
    projected = _curate_item_dict(item)
    assert len(projected["summary"]) == 150


def test_short_summary_not_padded():
    short_desc = "短描述"
    item = _make_item(short_desc)
    projected = _curate_item_dict(item)
    assert projected["summary"] == short_desc


# 验收 3: RSS 短摘要场景 < 30%
def test_ratio_rss_short_descriptions():
    """RSS 短摘要（~35 字符 description）100 条，精简比例 < 30%。"""
    desc = "人工智能在医疗领域取得新突破，GPT-5 展现卓越诊断能力。"
    items = [_make_item(desc, i) for i in range(100)]
    old = len(json.dumps([i.model_dump(mode="json") for i in items], ensure_ascii=False))
    new = len(json.dumps([_curate_item_dict(i) for i in items], ensure_ascii=False))
    ratio = new / old
    assert ratio < 0.30, f"RSS 短摘要场景精简比例 {ratio:.1%} 超过 30%"


# 验收 4: 混合分布场景 < 30%
def test_ratio_mixed_realistic_distribution():
    """混合分布（20%长+50%中+30%短）100 条，精简比例 < 30%。"""
    import random
    random.seed(42)
    descs_long = "这是一篇关于人工智能发展历史的深度长文。" * 30
    descs_mid = "<p>AI 技术持续发展，GPT-5 发布、Claude 升级。</p>"
    descs_short = "AI在医疗领域取得新突破。"
    items = []
    for i in range(100):
        r = random.random()
        if r < 0.2:
            desc = descs_long
        elif r < 0.7:
            desc = descs_mid
        else:
            desc = descs_short
        items.append(_make_item(desc, i))
    old = len(json.dumps([i.model_dump(mode="json") for i in items], ensure_ascii=False))
    new = len(json.dumps([_curate_item_dict(i) for i in items], ensure_ascii=False))
    ratio = new / old
    assert ratio < 0.30, f"混合场景精简比例 {ratio:.1%} 超过 30%"
```
