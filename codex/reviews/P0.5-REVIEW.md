## 4. 实现说明

- `SystemSettings` 通过 Pydantic `AliasChoices` 接收 `.env` 的无前缀变量和已有 `MULTISCRIBE_` 前缀变量；`model_validator(mode="after")` 将非空凭据单向复制到嵌套 Provider/Publisher 配置，空 API key 不会覆盖显式配置。
- `default_digest_targets` 与 `default_digest_adapter_ids` 使用 `NoDecode` 和 field validator 支持 `.env.example` 中的逗号分隔格式，避免 Pydantic Settings 将它当作 JSON 而失败。
- `ServiceContext.init()` 仅在 SQLite 尚无该 ID 时创建默认 AgentDefinition，使用现有领域模型的 `system_prompt` 字段，不覆盖用户配置，也不保存任何密钥。

## 5. 风险与后续

- P0.5 只接通设置与默认 Agent，不会提供真实外部凭据。P13 仍需决策者在本机 `.env` 配置有效 OpenAI 或 Anthropic key、飞书和企微 webhook，才能执行真实 e2e 并取得推送截图。
- P13 的 CLI `digest`、Docker、README 和 e2e 文件尚未实现，需在本包通过规划 review 后按原 P13 白名单重跑。
