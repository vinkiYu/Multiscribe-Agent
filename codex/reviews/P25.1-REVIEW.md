## 4. 详细任务完成情况

- **T1 上下文溢出保护**：`add_user()` 在历史裁剪前对当前消息按 80% 预算做头尾保留截断，增加 `[Truncated]` 标记；新增 `should_warn_budget()` 与 `estimated_tokens_remaining()`。
- **T2 工作流超时**：`stream()` 默认 300 秒，按 DAG level 使用剩余 deadline 包装 `asyncio.gather()`；超时 yield 带 `timeout` 字段的 `workflow_error`，嵌套工作流沿用剩余 timeout。
- **T3 Trace 传播**：新增 `inject_trace_headers()`、`trace_request_hook()` 和动态 SDK headers；Provider 工厂为可发现的 httpx client 安装 request hook，OTel 缺失或 span 无效时保持 no-op。
- **T4 速率限制**：新增按 IP、按规则 key 的滑动窗口 middleware；默认 auth 10/60s、agent 20/60s、digest 5/60s，支持实际路由别名和 `Retry-After`。

## 7. 风险、遗留与取舍

- **阻塞**：全仓库 format gate 被工作开始前已存在的 `daily_digest.py` 修改阻塞。该文件不在 P25.1 白名单内，不能由本包代为格式化。
- **Trace 范围**：本包按白名单接入 Provider 工厂和 Provider-owned httpx clients；其它插件适配器/发布器的独立 httpx client 未在本包范围内改动。
- **限流部署模型**：限流状态是进程内内存状态，多进程/多副本部署需要后续接入共享存储或网关限流。
- **OTel 依赖**：未安装 OTel SDK 时保持无损降级；本包没有新增可选依赖安装验证。
